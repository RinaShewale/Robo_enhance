Sahaay

AI Voice Assistant • Full Stack Documentation

Why This Project Was Built

Sahaay was built to simplify interaction using voice and text. It helps learners understand real-world full-stack development by combining frontend, backend, and database.

Project Overview

A full-stack voice assistant using Flask and PostgreSQL with UI + backend integration.

Installation & Run Commands


# 1. Go to backend folder
cd Robo_speaker/backend

# 2. Create virtual environment
python -m venv venv

# 3. Activate environment
# Windows:
venv\Scripts\activate
# Mac/Linux:
source venv/bin/activate

# 4. Install dependencies
pip install flask flask-cors psycopg2 python-dotenv

# 5. Run backend server
python app.py

Frontend (if separate folder):

cd frontend
python -m http.server 5500

Open browser at: http://127.0.0.1:5500

Database Structure

CREATE DATABASE Website_database;

CREATE TABLE app_users (
  id SERIAL PRIMARY KEY,
  username VARCHAR(100),
  email VARCHAR(100) UNIQUE,
  password_hash TEXT
);

CREATE TABLE chat_messages (
  id SERIAL PRIMARY KEY,
  user_id INTEGER REFERENCES app_users(id),
  message_content TEXT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE voice_interactions (
  id SERIAL PRIMARY KEY,
  user_input TEXT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

app_users: user data
chat_messages: chat storage
voice_interactions: voice logs

Backend Architecture

  • app.py → main server
  • auth.py → login/register
  • message.py → message handling
  • database.py → DB connection

Authentication APIs (Register & Login)

All requests must use JSON format in Thunder Client or curl.

Register User

Thunder Client POST request:

POST http://127.0.0.1:5000/register
{
  "username": "newuser",
  "email": "user@example.com",
  "password": "mypassword"
}

curl command:

curl -X POST http://127.0.0.1:5000/register \
-H "Content-Type: application/json" \
-d '{"username": "newuser", "email": "user@example.com", "password": "mypassword"}'

Login User

Thunder Client POST request:

POST http://127.0.0.1:5000/login
{
  "username": "newuser",
  "password": "mypassword"
}

curl command:

curl -X POST http://127.0.0.1:5000/login \
-H "Content-Type: application/json" \
-d '{"username": "newuser", "password": "mypassword"}'

Workflow

  1. User registers or logs in
  2. Frontend sends request to backend
  3. Backend processes & stores data
  4. Response sent back to UI
  5. User sees result on frontend
